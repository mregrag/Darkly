Vulnerability: Local File Inclusion (LFI)

Description:
The site routes pages through a single entry point, index.php, which
takes a "page" GET parameter to decide which file to include/display:

  index.php?page=signin
  index.php?page=survey
  index.php?page=member

The value of "page" is passed directly into a file-include mechanism
without proper validation or sanitization of path traversal sequences.

Exploitation:
By supplying a path traversal payload pointing outside the intended
web directory, arbitrary files readable by the web server can be
included/displayed. The number of "../" segments needed depended on
how deeply index.php sits relative to the filesystem root.

Payload used:
  index.php?page=../../../../../../../etc/passwd

Result:
The server included /etc/passwd, confirming arbitrary local file
read, and revealed the challenge flag.

Root cause:
User-controlled input is passed unsanitized into a file inclusion
function (e.g. PHP's include()/require()), allowing traversal outside
the intended directory and disclosure of arbitrary files on the
filesystem that the web server process can read.
